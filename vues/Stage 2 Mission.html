Analyse des besoins


BARACUI : utilisation d'un fichier Excel afin d'évaluer les applications nationales aux caisses de France.
Fichier placé à un endroit précis avec un nom précis, et ce, pour chaque fichier de chaque secteur.
Récupération de ces fichiers puis mise en forme avec de nombreuses macros.

Inventaire : l'inventaire n'étant pas à jour, des équipements se retrouve, dans GEDIFF (logiciel permettant le suivit de matériel), sans propriétaire alors que ce n'est pas le cas, ou simplement, le déplacement d'un agent d'un étage à l'autre qui n'a pas été enregistré dans le logiciel, ci qui peut causer des soucis de localisation.

RPA : (Robotic Process Automation). Beaucoup de personnes passent du temps à faire des copier-collers, ce qui leur faire perdre beaucoup de temps, est répétitif et peut être source d'erreur.


Problématiques


  • Créer une application web permettant aux managers de chaque secteur d'évaluer les applications nationales et récupérer celles-ci plus simplement.

  • Créer une application web affichant les anomalies détecté lors d'un inventaire (vérification de l'équipement).
    En combinaison avec un client lourd qui permet de scanner les étiquettes et comparer avec GEDIFF.

  • Créer des flux RPA à l'aide de Power Automate et automatiser les tâches.


Technologie utilisé


Symfony

bin : contient la console, executable et script pour le projet

config : contient les configurations des différents bundles, les routes, la sécurité et les services

public : contient le contrôleur frontal "index.php" et les "assets"

src : contient le projet (controleur, entités, répertoire, Form)

templates : contient les vues

var : contient les logs, le cache

vendor : contient les sources de bundles tiers
